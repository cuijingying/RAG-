# 培养方案智能问答系统技术报告

## 1. 项目背景与问题定义

### 1.1 项目背景

《非结构化数据处理》课程要求学生基于所学的大模型和 NLP 技术开发一个工程应用。本项目选择开发一个基于 RAG（Retrieval-Augmented Generation）技术的智能问答系统，旨在解决学生查询培养方案信息的需求。

### 1.2 问题定义

上海财经大学软件工程硕士研究生的培养方案包含大量的课程信息、学分要求、修读规则等，目前学生需要手动查阅 PDF 文档或询问教务人员，效率较低。本项目需要开发一个智能问答系统，支持：

- 自然语言查询课程信息、学分要求、修读规则等
- 基于语义理解的智能检索
- 准确、实时的问答响应
- 友好的交互界面

### 1.3 技术挑战

1. **文档结构化处理**：培养方案文档包含复杂的 Markdown 结构，需要合理切片
2. **语义理解**：需要准确理解用户问题的语义意图
3. **知识库构建**：需要构建高效的向量数据库支持语义检索
4. **流式响应**：需要实现实时的流式问答提升用户体验
5. **API 集成**：需要集成 DeepSeek API 实现大模型调用

## 2. 核心算法与技术栈

### 2.1 技术栈选择

| 类别 | 技术 | 原因 |
|------|------|------|
| 前端框架 | Streamlit | 快速构建交互式 Web 应用，适合数据科学项目 |
| RAG 框架 | LangChain | 提供完整的 RAG 工具链，支持多种向量数据库 |
| 向量数据库 | Chroma | 轻量级、开源、支持本地存储 |
| 大模型 API | DeepSeek API | 性价比高、兼容 OpenAI 接口 |
| 文本嵌入 | BAAI/bge-small-zh-v1.5 | 中文语义检索效果好、轻量级 |
| 容器化 | Docker | 支持一键部署、跨平台运行 |

### 2.2 核心算法实现

#### 2.2.1 文档切片算法

**算法选择**：MarkdownHeaderTextSplitter

**实现细节**：
```python
from langchain_text_splitters import MarkdownHeaderTextSplitter

headers_to_split_on = [
    ("##", "Header 2")
]
markdown_splitter = MarkdownHeaderTextSplitter(headers_to_split_on=headers_to_split_on)
docs = markdown_splitter.split_text(markdown_text)
```

**算法优势**：
- 保留文档的语义结构
- 按二级标题（##）切片，确保每个切片包含完整的课程模块信息
- 自动提取元数据（Header 2），便于后续追踪文档来源

**切片结果**：
- 共生成 10 个文档片段
- 每个片段对应培养方案的一个主要模块（学位公共课、学位基础课、专业必修课等）

#### 2.2.2 向量嵌入算法

**算法选择**：BAAI/bge-small-zh-v1.5

**模型参数**：
- 模型大小：~400MB
- 向量维度：512
- 语言：中文优化

**实现细节**：
```python
from langchain_huggingface import HuggingFaceEmbeddings

embeddings = HuggingFaceEmbeddings(
    model_name="BAAI/bge-small-zh-v1.5",
    model_kwargs={'device': 'cpu'},
    encode_kwargs={'normalize_embeddings': True}
)
```

**算法优势**：
- 专为中文语义检索优化
- 轻量级，适合本地部署
- 支持向量归一化，提升检索准确度

#### 2.2.3 相似度检索算法

**算法选择**：Chroma 向量数据库 + 余弦相似度

**实现细节**：
```python
from langchain_chroma import Chroma

vectorstore = Chroma.from_documents(
    documents=docs,
    embedding=embeddings,
    persist_directory="./chroma_db"
)

retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
docs = retriever.invoke(query)
```

**算法特点**：
- 基于向量嵌入的语义相似度检索
- 检索 Top-3 最相关文档片段
- 支持持久化存储，无需重复构建

#### 2.2.4 检索增强生成（RAG）

**算法流程**：
1. 用户提问 → 向量嵌入
2. 向量数据库检索 → 相关文档片段
3. 组装 Prompt（用户问题 + 检索上下文）
4. 大模型生成回答

**Prompt 设计**：
```python
template = """你是一个专业的学术顾问，专门回答关于上海财经大学软件工程专业硕士研究生培养方案的问题。

请根据以下检索到的相关文档片段，准确回答用户的问题。

【检索到的相关文档片段】：
{context}

【用户问题】：
{question}

回答要求：
1. 基于提供的文档片段回答，不要编造信息
2. 如果文档中没有相关信息，请明确说明
3. 回答要准确、清晰、有条理
4. 可以适当引用具体的课程代码、学分、学期等信息
5. 使用友好、专业的语气

请开始回答："""
```

**算法优势**：
- 结合检索上下文，提高回答准确性
- 明确要求基于文档回答，避免幻觉
- 支持流式输出，提升用户体验

### 3.2 核心模块实现

#### 3.2.1 向量数据库构建模块（ingest.py）

**功能**：读取培养方案文档，切片、嵌入、存储到向量数据库

**关键代码**：
```python
def main():
    # 1. 读取文档
    with open("program.md", "r", encoding="utf-8") as f:
        markdown_text = f.read()

    # 2. 切片
    docs = markdown_splitter.split_text(markdown_text)

    # 3. 嵌入与存储
    vectorstore = Chroma.from_documents(
        documents=docs,
        embedding=embeddings,
        persist_directory="./chroma_db"
    )
```

**特点**：
- 支持增量更新
- 自动持久化
- 提供切片预览

#### 3.2.2 问答应用模块（app.py）

**功能**：提供交互式问答界面，实现 RAG 检索和流式回答

**关键功能**：
1. **缓存机制**：使用 `@st.cache_resource` 缓存加载的模型和数据库
2. **流式输出**：使用 `stream=True` 实现打字机效果
3. **上下文展示**：可展开查看检索到的文档片段
4. **对话历史**：支持多轮对话

**关键代码**：
```python
@st.cache_resource
def load_vector_store():
    return Chroma(
        persist_directory="./chroma_db",
        embedding_function=get_embeddings()
    )

# 流式回答
response = llm.stream(formatted_prompt)
for chunk in response:
    if chunk.content:
        full_response += chunk.content
        message_placeholder.markdown(full_response + "▌")
```

## 5. 项目局限性

### 5.1 技术局限性

1. **向量嵌入质量**：使用开源中文 Embedding 模型，语义理解能力有限
2. **检索准确性**：基于余弦相似度的检索，可能遗漏相关文档
3. **大模型依赖**：依赖 DeepSeek API 的稳定性和成本
4. **知识库规模**：当前仅包含 1 个培养方案文档，可扩展性有限

### 5.2 功能局限性

1. **多轮对话能力**：不支持复杂的多轮上下文对话
2. **推理能力**：仅支持基于检索信息的直接回答，不支持复杂推理
3. **更新机制**：文档更新需要手动重新构建向量库
4. **用户权限**：缺少用户身份验证和权限管理

### 5.3 部署局限性

1. **单机部署**：当前不支持分布式部署
2. **并发限制**：Streamlit 的并发处理能力有限
3. **资源占用**：Embedding 模型占用 ~400MB 内存

## 6. 改进方向

### 6.1 技术改进

1. **优化检索算法**：引入重排序（Re-ranking）机制提升检索准确性
2. **改进 Embedding 模型**：使用更大的中文语义模型（如 bge-large-zh）
3. **增强上下文理解**：支持多文档综合检索和答案生成
4. **优化 Prompt 工程**：引入思维链（Chain-of-Thought）提升推理能力

### 6.2 功能扩展

1. **多文档支持**：支持多个培养方案文档的统一检索
2. **智能推荐**：基于用户历史查询推荐相关问题
3. **数据统计**：提供查询统计和热门问题分析
4. **导出功能**：支持导出问答记录和培养方案摘要

### 6.3 性能优化

1. **缓存优化**：增加查询结果缓存，减少重复计算
2. **并行检索**：支持多个检索器并行查询
3. **模型量化**：使用量化后的 Embedding 模型减少内存占用
4. **分布式部署**：支持向量数据库的分布式部署

## 7. 总结

本项目成功实现了一个基于 RAG 技术的培养方案智能问答系统，完整应用了《非结构化数据处理》课程所学的核心技术：

- **文档处理**：使用 MarkdownHeaderTextSplitter 实现结构化文档切片
- **向量嵌入**：使用 BAAI/bge-small-zh-v1.5 模型生成语义向量
- **相似度检索**：基于 Chroma 向量数据库实现语义检索
- **大模型应用**：集成 DeepSeek API 实现智能问答
- **工程化能力**：使用 Streamlit + Docker 实现快速开发和部署

项目具有良好的可扩展性和鲁棒性，能够满足基本的学生问答需求。同时，项目也明确了当前的技术局限性和未来的改进方向，为后续优化提供了清晰的路线图。
